## 📖 Overview

**VastuResha** is a premium, single-page demo website for a boutique architecture and interiors studio based in Chinchwad, Pune — led by **Ar. Reshma Wanjari**. The site fuses traditional Vastu Shastra philosophy with a refined, editorial visual language.

Built with zero dependencies. No frameworks. No build tools. Just clean HTML, CSS, and vanilla JS — ready to deploy anywhere in seconds.

---

## 🖥️ Live Preview

> Open `vasturesha-demo-site.html` directly in any browser — no server required.

```bash
# Clone the repo
git clone https://github.com/your-username/vasturesha-demo-site.git

# Open in browser
open vasturesha-demo-site.html
```

## ✨ Features

- **Zero dependencies** — pure HTML5, CSS3, Vanilla JS
- **Fully responsive** — mobile, tablet, and desktop layouts
- **Smooth scroll navigation** with active section awareness
- **Frosted-glass sticky header** that transitions on scroll
- **CSS entry animations** — orchestrated `riseIn` hero sequence
- **Animated scroll indicator** with looping line cue
- **Jali lattice motif** — SVG dot-grid texture referencing traditional Indian screens
- **Accessible** — respects `prefers-reduced-motion`, semantic HTML throughout
- **Google Fonts** — Fraunces (serif display) + Archivo (body) pairing

## 🎨 Design System

### Colour Palette

| Name | Hex | Role |
|---|---|---|
| Limestone | `#F5F1EA` | Page background |
| Ink | `#1C1A17` | Primary text & borders |
| Teak | `#8B7355` | Accent colour, eyebrows |
| Brass | `#C9A876` | Highlight details |
| Graphite | `#3A3835` | Secondary / muted text |

### Typography

| Role | Typeface | Weight |
|---|---|---|
| Display / Headings | Fraunces (serif) | 300 – 600 |
| Body | Archivo | 400 – 600 |
| Labels / Eyebrows | Archivo Expanded | 500 – 700 |

### Signature Element
The **Jali dot-grid** — a `radial-gradient` dot pattern used as a translucent texture across hero and portrait sections, echoing the traditional Indian *jali* (perforated screen) motif.

## 🚀 Deployment

Works out of the box on any static host:

| Platform | Command / Steps |
|---|---|
| **GitHub Pages** | Push to `main` → enable Pages from repo Settings |
| **Netlify** | Drag & drop the HTML file at netlify.com/drop |
| **Vercel** | `vercel --prod` in the repo directory |
| **Any server** | Upload the single `.html` file |
